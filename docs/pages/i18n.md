# i18n

**i18n is not part of the native client Swift SDK.** There is no `t()`,
label-render helper, or translation-loader in this package — translation
*rendering* is a browser-SDK / SSR feature, not something a native app needs from
Shipeasy.

A native iOS / macOS / tvOS / watchOS app localizes its user-facing copy with the
platform's own tooling: **String Catalogs** (`.xcstrings`) or `Localizable.strings`
via `NSLocalizedString` / SwiftUI's `Text` and `LocalizedStringKey`. That is the
right home for on-device localization, and it works fully offline.

This SDK covers feature flags ([flags](flags.md)), dynamic configs
([configs](configs.md)), kill switches ([killswitches](killswitches.md)),
A/B experiments ([experiments](experiments.md)), and `see()` error reporting
([error-reporting](error-reporting.md)) — no i18n API is exposed or required.
